No Table Description
01 users Stores core user account information, authentication credentials, and user roles.

- Primary keys: id
- Foreign keys: None
02 provider_profiles Detailed profile information for Food Providers (e.g., restaurants, supermarkets).

- Primary keys: id
- Foreign keys: user_id (The associated user account)
03 receiver_profiles Detailed profile information for Receivers (individuals needing food assistance).

- Primary keys: id
- Foreign keys: user_id (The associated user account)
04 organization_profiles Detailed profile information for Charity Organizations.

- Primary keys: id
- Foreign keys: user_id (The associated user account)
05 organization_members Stores members or beneficiaries managed by a specific charity organization.

- Primary keys: id
- Foreign keys: organization_id (The organization they belong to)
06 group_pickups Manages bulk or group food pickups scheduled by charity organizations.

- Primary keys: id
- Foreign keys: organization_id (The organizing charity), reservation_id (The associated bulk reservation)
07 volunteer_profiles Detailed profile information for Volunteers, including availability and rank.

- Primary keys: id
- Foreign keys: user_id (The associated user account)
08 volunteer_specializations Stores specific skills, roles, or certifications for volunteers (e.g., shipper, chef).

- Primary keys: id
- Foreign keys: volunteer_id (The volunteer who owns the specialization)
09 liability_waivers Records user acceptances of system liability and safety waivers.

- Primary keys: id
- Foreign keys: user_id (The user who accepted the waiver)
10 refresh_tokens Stores active JWT refresh tokens to maintain user sessions securely.

- Primary keys: id
- Foreign keys: user_id (The user owning the session)
11 device_tokens Stores FCM device tokens used for sending push notifications.

- Primary keys: id
- Foreign keys: user_id (The user who owns the device)
12 verification_requests Stores requests and documents submitted by users to verify their accounts.

- Primary keys: id
- Foreign keys: user_id (The user requesting verification)
13 food_listings Stores details of surplus food items offered by providers.

- Primary keys: id
- Foreign keys: provider_id (The provider creating the listing)
14 reservations Records food bookings/orders made by receivers or organizations.

- Primary keys: id
- Foreign keys: listing_id (The reserved food item), receiver_id (The receiver, if individual), organization_id (The organization, if bulk)
15 deliveries Manages the logistics and status of delivering reserved food by volunteer shippers.

- Primary keys: id
- Foreign keys: reservation_id (The reservation being delivered), shipper_id (The assigned volunteer shipper)
16 shipper_task_offers Stores delivery task offers broadcasted to nearby available volunteer shippers.

- Primary keys: id
- Foreign keys: delivery_id (The delivery task), shipper_id (The volunteer receiving the offer)
17 kitchen_campaigns Manages community kitchen events and campaigns organized by charities.

- Primary keys: id
- Foreign keys: organization_id (The organizing charity)
18 campaign_change_requests Stores requests to modify the details or schedule of an existing kitchen campaign.

- Primary keys: id
- Foreign keys: campaign_id (The campaign to modify), requested_by_user_id (The user requesting the change)
19 campaign_donations Records ingredient donations pledged by providers for a kitchen campaign.

- Primary keys: id
- Foreign keys: campaign_id (The destination campaign), provider_id (The donating provider)
20 campaign_volunteer_assignments Assigns specific roles (chef, waiter, etc.) to volunteers in a campaign.

- Primary keys: id
- Foreign keys: campaign_id (The campaign), volunteer_id (The assigned volunteer), shift_id (The specific shift)
21 campaign_shifts Defines specific time slots and role requirements for a kitchen campaign.

- Primary keys: id
- Foreign keys: campaign_id (The associated campaign)
22 campaign_experiences Stores feedback, ratings, and experiences shared by volunteers after a campaign.

- Primary keys: id
- Foreign keys: campaign_id (The campaign), volunteer_id (The volunteer sharing the experience)
23 recipes Stores standardized cooking recipes created by users.

- Primary keys: id
- Foreign keys: created_by_user_id (The creator of the recipe)
24 recipe_ingredients Lists the specific ingredients and quantities required for a recipe.

- Primary keys: id
- Foreign keys: recipe_id (The associated recipe)
25 campaign_menu_items Associates specific recipes to form the menu of a kitchen campaign.

- Primary keys: id
- Foreign keys: campaign_id (The campaign), recipe_id (The selected recipe)
26 kitchen_safety_logs Records food safety, temperature, and hygiene checks during a kitchen campaign.

- Primary keys: id
- Foreign keys: campaign_id (The campaign), checked_by_volunteer_id (The volunteer performing the check)
27 meal_distributions Tracks the distribution rounds of cooked meals during a campaign.

- Primary keys: id
- Foreign keys: campaign_id (The campaign), served_by_volunteer_id (The volunteer distributing the meals)
28 meal_feedback Stores feedback and satisfaction ratings from beneficiaries regarding the distributed meals.

- Primary keys: id
- Foreign keys: distribution_id (The specific distribution round)
29 ratings Stores ratings and reviews given between users (e.g., receiver rating provider).

- Primary keys: id
- Foreign keys: rater_id (The user giving the rating), ratee_id (The user receiving the rating)
30 trust_score_history Logs all changes (penalties/bonuses) to users' trust scores over time.

- Primary keys: id
- Foreign keys: user_id (The user whose score changed)
31 dedication_points_history Logs all changes to volunteers' dedication points over time.

- Primary keys: id
- Foreign keys: volunteer_id (The volunteer whose points changed)
32 reports Stores violation or issue reports submitted by users against others or listings.

- Primary keys: id
- Foreign keys: reporter_id (The user reporting the issue)
33 notifications Stores system notifications sent to individual users.

- Primary keys: id
- Foreign keys: user_id (The recipient of the notification)
34 audit_logs Records system-level actions and changes for auditing and security purposes.

- Primary keys: id
- Foreign keys: actor_id (The user performing the action, if applicable)
35 system_configs Stores dynamic global system configurations (key-value pairs).

- Primary keys: id
- Foreign keys: None
36 esg_snapshots Stores periodic snapshots of Environmental, Social, and Governance (ESG) metrics.

- Primary keys: id
- Foreign keys: provider_id (The provider, if applicable), organization_id (The organization, if applicable)
